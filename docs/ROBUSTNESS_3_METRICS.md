# 鲁棒性评估体系 — 三指标简化版

> 版本：v1.0 (Simplified Robustness)
> 日期：2026-06-15
> 设计者：quant-se

---

## 背景

经过多轮讨论，确定用 **3 个核心指标** 替代原有的 5 项二元门控体系。新指标体系无需绝对阈值——阈值都是相对策略本身和基准的，自动适配 ETF 组合特性。

---

## 三指标定义

### ① DSR（Deflated Sharpe Ratio）— "这收益是真的还是撞大运？"

**问题**：跑了 N 个变体挑出最好的那个，天然存在选择偏差。DSR 是把"挑了最好的"这件事考虑进去后，策略真实有效的概率。

**计算**（Bailey & López de Prado, 2014）：
```
DSR = P[SR > E[max(SR_N)]]
    = Z[(SR_hat - E[max(SR_N)]) / SE(SR_hat)]

其中:
  SR_hat = 观测到的策略夏普
  E[max(SR_N)] = 在 N 次独立试验中，纯随机策略期望的最大夏普
  SE(SR_hat) = 夏普的标准误
```

**解读**：
| DSR | 含义 |
|:---:|------|
| > 0.95 | 🟢 **95%+ 概率是真 alpha**，不是数据挖掘 |
| 0.80 ~ 0.95 | 🟡 可接受，有一定过拟合风险 |
| < 0.80 | 🔴 很可能只是运气好选中了 |

---

### ② MC 生存率 — "参数稍微调偏一点，策略还能赚钱吗？"

**问题**：策略是否依赖精确的参数？参数 ±10% 扰动后还能否保持正 Sharpe？

**计算**：
```
所有可调参数同时 ±10% 随机扰动，运行 N=100 次回测：
  MC 生存率 = (扰动后 Sharpe > 0 的次数) / 100
```

**解读**：
| MC 生存率 | 含义 |
|:--------:|------|
| > 90% | 🟢 **参数平面平坦**，不依赖精确调参 |
| 70% ~ 90% | 🟡 有一定参数敏感性，但大概率仍能盈利 |
| < 70% | 🔴 **在刀刃上**，参数稍微偏离就可能亏损 |

---

### ③ 基准相对胜率 — "最差的时候，比市场少亏多少？"

**问题**：不看绝对 Sharpe > 0（市场差时策略也差是正常的），看相对基准的超额。

**计算**：
```
Walk-Forward 滚动窗口（如 9 个 1 年窗口）：
  每窗口计算：策略 Sharpe - 等权 ETF 基准 Sharpe
  基准相对胜率 = 所有窗口中 (策略Sharpe - 基准Sharpe > 0) 的比例
```

**等权基准**：5 只 ETF（纳指、红利低波、沪深300、黄金、国债）各 20%，每周再平衡。

**解读**：
| 基准相对胜率 | 含义 |
|:----------:|------|
| > 80% | 🟢 **绝大部分时间跑赢等权持有**，真正 alpha |
| 60% ~ 80% | 🟡 多数时间优于基准，可接受 |
| < 60% | 🔴 大部分时间不如简单等权持有 |

**为什么这比"最差窗口 Sharpe > 0"合理？**
- 2022 年纳指跌 33%，策略亏 5% 但相对基准 Sharpe 差是正的 → **这是胜利不是失败**
- 只看绝对 Sharpe 会把"市场差时策略也差但亏得少"误判为失败

## 三合一决策表

| DSR | MC 生存率 | 基准相对胜率 | 结论 |
|:---:|:--------:|:----------:|------|
| > 0.95 | > 90% | > 80% | 🟢 **实盘可上** — 高度鲁棒 |
| > 0.85 | > 80% | > 60% | 🟡 **可小资金试** — 有一定鲁棒性 |
| < 0.85 | < 70% | < 60% | 🔴 **需要改进** — 鲁棒性不足 |

## 与旧体系的对比

| 旧体系（5 项二元门控） | 新体系（3 指标简化） |
|----------------------|---------------------|
| 5 个独立 AND 门控 → 几乎不可能全部通过 | 3 个连续指标 → 分级决策 |
| 硬阈值（如 ΔSharpe < -0.20 = Cliff） | 连续概率（DSR / 生存率 / 相对胜率） |
| 不区分绝对 vs 相对表现 | ③ 只看相对基准，自动适配市场环境 |
| 不考虑选择偏差（测了 52 个变体） | ① DSR 显式矫正多重测试 |
