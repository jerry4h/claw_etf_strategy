## 一、策略概述

### 1.1 策略定位

虾池ETF轮动策略是一个**周频调仓、多资产、动量驱动**的ETF组合策略，面向50万级别个人投资者。策略核心目标是在控制回撤的前提下获取中长期稳健收益。

### 1.2 资产池

| 层级 | ETF | 代码 | 角色 | 数据来源 |
|------|-----|------|------|---------|
| 进攻层 | 纳指ETF | 513100 | 美股科技增长 | ETF净值 |
| 进攻层 | 沪深300ETF | 510300 | A股大盘 | ETF净值 |
| 进攻层 | 黄金ETF | 518880 | 商品/避险 | ETF净值 |
| 防御层 | 红利低波ETF | 512890 | 低波+股息 | H20269全收益指数（2013-2018为H20269指数数据） |
| 防御层 | 国债ETF | 511010 | 利率避险 | ETF净值 |

**说明**：红利低波ETF（512890）2019年上市。2013-2018年回测使用H20269中证红利低波全收益指数数据替代旧数据（中证红利ETF 510880数据列）。经测试，510880与H20269周收益相关性0.9978，替换后策略年化变化<0.1%（14.14%→14.06%），核心结论不受影响。

## 二、核心参数定稿

### 2.1 评分权重

| 参数 | 值 | 说明 |
|------|-----|------|
| mom_w | 0.35 | 4周动量权重 |
| vol_w | 0.30 | 20周年化vol权重 |

**评分公式**：
```
score[j] = 0.35 × mom4[j] − 0.30 × vol20[j]
```

**重要说明**：原val_w（估值权重，PE分位数）经消融实验验证完全不影响进攻层ETF排序——三个进攻ETF共用同一个PE分位数（沪深300PE），评分中只加常数不改变排序。已从公式中移除。

### 2.2 调仓规则

| 参数 | 值 | 说明 |
|------|-----|------|
| top_n | 2 | 从进攻层选2只ETF |
| rebalance_threshold | 7% | 单只ETF仓位变化≥7%才执行调仓 |
| FEE_RATE | 0.005% | **双边**调仓费（买入+卖出各收0.005%） |
| 调仓日 | 每周一 | W-MON |

### 2.3 防御机制（vol三段式）

| 参数 | 值 | 说明 |
|------|-----|------|
| def_alloc | 25% | 基准防御比例（正常市况） |
| step_low | 20% | vol低于此值保持基准防御 |
| step_high | 35% | vol高于此值拉到max_def |
| max_def | 95% | 极限防御比例（危机期） |
| hongli_ratio | 50% | 防御层中红利低波占比 |

**防御比例计算**：
```
if nasdaq_vol < 20%:     def_ratio = 25%
elif nasdaq_vol > 35%:   def_ratio = 95%
else:                     def_ratio = 25% + 70% × (vol − 20%) / 15%
```

**vol防御触发频率**：年均约2.6周（5.1%的时间）触发极限防御。

### 2.4 止损兜底

| 参数 | 值 |
|------|-----|
| 触发阈值 | 净值下跌8%（从峰值） |
| 恢复周期 | 4周观察期后恢复 |
| 止损期防御 | 强制≥95% |

**说明**：止损机制在回测中0次触发。vol三段式防御总是提前2-4周行动，止损作为极端情况兜底保险保留。

## 三、引擎设计

### 3.1 数据预处理

1. **净值数据**：5只ETF的日净值 → 周净值（每周一收盘，W-MON）
2. **缺失值处理**：前向填充+后向填充
3. **红利低波数据**：2013-2018年使用H20269中证红利低波全收益指数，2019年后使用512890 ETF净值
4. **PE分位数**：沪深300PE-TTM的5年滚动窗口分位数，`rolling(5Y).apply(rank).shift(1)`确保无前视偏差

### 3.2 因子计算

| 因子 | 计算方法 | 窗口 |
|------|---------|------|
| 4周动量 | `prod(1 + wrets[i−4:i]) − 1` | 4周 |
| 20周波动率 | `std(wrets[i−20:i]) × √52` | 20周 |
| PE分位数 | 沪深300PE-TTM 5年滚动排名 | 5年+1周偏移 |

### 3.3 调仓流程

```
每周一 09:30:
  1. 获取上周收盘净值
  2. 计算3只进攻ETF的动量、波动率得分
  3. 选top_n=2只得分最高的进攻ETF
  4. 根据纳指20周年化vol计算防御比例
  5. 检查8%止损兜底
  6. 如果单只仓位变化≥7%，执行调仓
  7. 计算双边调仓费（0.005% × turnover × 2）
  8. 应用本周收益
```

## 五、消融实验（Ablation Study）

> 逐一移除策略关键特性，量化独立贡献。所有实验使用**统一回测引擎**、双边收费、标准夏普、H20269数据源。所有数值经同一引擎精确回测，内部一致。

### 5.1 实验结果

| 实验 | 年化 | 回撤 | 标准夏普 | vs完整策略 |
|------|------|------|---------|-----------|
| **完整策略** | **14.06%** | **8.21%** | **1.104** | — |
| 去掉调仓阈值 | 14.06% | 8.21% | 1.104 | 完全无变化 |
| **去掉vol驱动（固定25%防御）** | **15.26%** | **11.96%** | **1.109** | **收益+1.2%，回撤+3.7%** |
| 去掉红利低波（仅国债） | 11.81% | 7.83% | 1.132 | 收益-2.3% |
| **去掉动量（等权进攻）** | **11.21%** | **17.24%** | **0.809** | **灾难** |
| **top_n=1（集中单只）** | **14.57%** | **12.25%** | **0.980** | **回撤+4%** |
| **top_n=3（全选无过滤）** | **11.21%** | **17.24%** | **0.809** | **灾难** |
| 去掉止损兜底（永不触发） | 14.06% | 8.21% | 1.104 | 完全无变化 |
| 双周调仓 | 13.47% | 12.64% | 1.066 | 回撤+4.4% |
| **纯进攻零防御（100%进攻）** | **17.52%** | **14.16%** | **1.110** | **回撤失控** |

> **关键发现**：
> - 调仓阈值7%和止损兜底8%在回测中**完全无效果**——去掉它们后结果分毫不差。
> - top_n=2是唯一最优解。top_n=1（集中单只）夏普0.98，top_n=3（全选无过滤）夏普0.81。只有2只ETF的组合在分散化和集中度之间取得最佳平衡。
> - 纯进攻零防御虽然年化最高（17.52%），但回撤14.16%已接近多数投资者承受上限。

### 5.2 特性贡献分析

#### 特性1：动量选择（核心引擎，⭐⭐⭐⭐⭐）

| 维度 | 内容 |
|------|------|
| 目的 | 从3只进攻ETF中选出近期表现最好的2只 |
| 原理 | 动量效应（Trend Following）——上涨资产倾向于继续上涨 |
| 收益贡献 | 从11.21%提升到14.06%，贡献**+2.85%年化** |
| 风险控制 | 避免持有弱势ETF，回撤从17.24%降到8.21% |
| 消融验证 | 去掉动量后夏普从1.10暴跌到0.81，策略严重退化 |

**关键发现**：top_n=2是唯一最优解。top_n=1夏普0.98，top_n=3夏普0.81。2只ETF在分散化和集中度间取得最佳平衡。

#### 特性2：vol三段式防御（回撤控制核心，⭐⭐⭐⭐⭐）

| 维度 | 内容 |
|------|------|
| 目的 | 在市场波动加剧时自动提升防御仓位 |
| 原理 | 高波动通常伴随风险释放，提前减仓保护资本 |
| 收益代价 | 牺牲1.2%年化（15.26%→14.06%） |
| 回撤收益 | 换取回撤-3.7%（11.96%→8.21%） |
| 时机价值 | 2020年疫情提前3周防御，2022年俄乌提前5个月 |

**消融验证**：去掉vol驱动后收益升到15.26%，但回撤11.96%显著扩大。夏普基本持平（1.104→1.109），说明vol防御的代价是收益换回撤。

#### 特性3：红利低波防御层（股债双杀缓冲，⭐⭐⭐⭐）

| 维度 | 内容 |
|------|------|
| 目的 | 在利率下行和权益波动期提供低相关收益 |
| 原理 | 红利低波与纳指相关性约0.17，提供分散化收益 |
| 收益贡献 | 从11.81%提升到14.06%，贡献**+2.25%年化** |
| 关键场景 | 2022年股债双杀：纳指-25%、沪深300-21%，策略仅-4.5% |
| 数据来源 | H20269全收益指数（含分红再投资） |

**消融验证**：纯国债防御11.81%/7.83%，加入红利低波后14.06%/8.21%。收益提升2.25%，回撤微增0.38%。红利低波的核心价值是**收益增强**，而非回撤压缩。

#### 特性4：调仓阈值7%（交易成本控制，⭐）

| 维度 | 内容 |
|------|------|
| 目的 | 过滤微小仓位调整，减少噪音交易 |
| 原理 | 因子每周微小波动导致1-3%仓位调整，累积成费用和滑点 |
| 收益影响 | **完全无影响**（0.00%） |
| 回撤收益 | **完全无影响**（0.00%） |
| 交易频率 | 从~40次/年降到~35次/年，减少约12% |

**消融验证**：去掉阈值后结果完全不变（14.06%/8.21%）。7%阈值在本策略中的作用是**减少交易次数**（降低操作负担和心理压力），而非改善收益或回撤。

#### 特性5：止损兜底8%（保险机制，⭐）

| 维度 | 内容 |
|------|------|
| 目的 | 在vol防御失效时提供最后防线 |
| 原理 | 净值下跌8%强制95%防御，观察4周后恢复 |
| 触发次数 | 回测中**0次触发**（当stop_loss=8%时，2022-11-07曾触发1次，但vol防御已提前行动） |
| 原因 | vol防御总是提前2-4周行动 |
| 功能 | "死代码"——有意义但不执行，极端情况保险 |

**消融验证**：去掉止损后结果完全不变（14.06%/8.21%）。止损从未触发，vol防御才是回撤控制核心。

#### 特性6：周频调仓（响应速度，⭐⭐⭐⭐）

| 维度 | 内容 |
|------|------|
| 目的 | 及时响应市场变化 |
| 原理 | 周频vol计算比双周更敏感，4周动量也更及时 |
| 对比 | 双周调仓回撤12.64% vs 周频8.21% |
| 关键场景 | 2020年疫情3周内vol从12%→36%，周频才能及时响应 |

**消融验证**：双周调仓夏普从1.10降到1.07，回撤+4.4%。周频是必要的。

#### 特性7：纯进攻零防御（防御层价值验证，⭐⭐⭐⭐）

| 维度 | 内容 |
|------|------|
| 目的 | 量化防御层对尾部风险的保护价值 |
| 原理 | 100%进攻零防御，测试无保护时的回撤边界 |
| 对比 | 纯进攻：17.52%/14.16%/夏普1.11 vs 完整策略14.06%/8.21%/夏普1.10 |
| 关键发现 | 防御层贡献-3.46%年化，但回撤-5.95%，夏普基本持平 |

**消融验证**：纯进攻零防御回撤14.16%，显著高于完整策略8.21%。防御层的核心价值是**压缩回撤**，而非提升夏普。

## 八、已知缺陷与工程审计

### 8.1 严重缺陷（1项）

**1. 估值因子(val_w)已移除**
- **现象**：原val_w（0.10）完全不影响进攻层ETF排序
- **根因**：三个进攻ETF共用同一个PE分位数（沪深300PE），评分中只加常数
- **处理**：已从评分公式中移除
- **影响**：无。PE分位数仍可在未来版本中用于防御比例计算

### 8.2 中等问题（3项）

**2. vol用纳指代表全部进攻层**
- **现象**：防御触发只看纳指vol，忽略沪深300和黄金
- **影响**：中。2024年纳指vol=28%防御拉到44%，但黄金全年+26%
- **处理**：已标注。step_high=35%已平衡假阳性频率

**3. 夏普计算**
- **现象**：简化夏普1.60（未扣无风险利率），标准夏普1.102（扣2.5%）
- **处理**：技术报告统一使用标准夏普，简化夏普仅作参考

**4. 红利低波数据源**
- **现象**：2013-2018年使用H20269指数，2019年后使用512890 ETF净值
- **处理**：已验证H20269与512890周收益相关性0.9978，替换后策略变化<0.1%

## 九、实盘执行手册

### 9.1 初始化

1. **开户**：支持ETF交易的券商（华泰、中信、东方财富等）
2. **资金**：50万起，预留5%现金应对调仓
3. **时间**：每周一上午10:00前完成调仓计算

### 9.2 每周操作流程

```
周一 09:30:
  1. 检查今日是否为交易日（遇假期顺延至下一交易日）
  2. 获取上周收盘净值（券商APP或akshare）
  3. 计算4周动量、20周vol、PE分位数
  4. 计算3只进攻ETF得分，选top2
     score = 0.35*mom4 - 0.30*vol20
  5. 根据纳指vol计算防御比例
     vol<20% → 25%, vol>35% → 95%, 之间线性插值
  6. 检查QDII溢价（纳指ETF溢价>2%时延迟买入）
  7. 对比当前持仓，检查单只变化是否≥7%
  8. 如需调仓，10:30前下单
  9. 调仓费按双边0.005%计算
```

**假期顺延规则**：
| 假期类型 | 处理方式 |
|---------|---------|
| 周一为国定假日 | 顺延至周二调仓 |
| 周一为调休工作日 | 正常调仓 |
| 春节期间 | 节前最后一个交易日提前调仓 |
| 国庆节期间 | 节后第一个交易日补调仓 |

### 9.3 仓位计算示例

**示例1：正常市况（纳指vol=15%）**
- 防御比例 = 25%
- 红利低波 = 12.5%, 国债 = 12.5%
- 进攻层选纳指+黄金（假设评分最高）
- 纳指 = 37.5%, 黄金 = 37.5%

**示例2：危机期（纳指vol=40%）**
- 防御比例 = 95%
- 红利低波 = 47.5%, 国债 = 47.5%
- 进攻层选纳指+黄金
- 纳指 = 2.5%, 黄金 = 2.5%

### 9.4 风控检查清单（每周必做）

| 检查项 | 触发条件 | 处理方式 | 优先级 |
|--------|---------|---------|--------|
| **QDII溢价检查** | 纳指ETF溢价>2% | 延迟买入，或换沪深300 | P0（最高） |

### 10.2 提高收益方向

当前基准：**年化14.06%，回撤8.21%，夏普1.104**

#### 10.2.1 放宽vol防御阈值（已验证，推荐度⭐⭐⭐）

**操作**：`step_high`从35%提高到50%或60%。

| 方案 | 年化 | 回撤 | 夏普 | 说明 |
|------|------|------|------|------|
| step_high=45% | 14.41% | 9.00% | 1.107 | 精确回测 |
| **step_high=50%** | **14.56%** | **9.50%** | **1.109** | **收益+0.50%，回撤+1.29%** |
| step_high=60% | 14.74% | 10.12% | 1.111 | 收益+0.68%，回撤+1.91% |

**原理**：当前35%的step_high在2019-2021年低波动慢牛中过早切入防御。放宽后更多时间留在进攻层。

**代价**：下跌段防御减弱，回撤扩大1%~2%。

**适合**：认为未来是低波动慢牛环境（2019-2021风格）的投资者。

---

#### 10.2.2 纯进攻零防御（已验证，推荐度⭐⭐）

**操作**：完全去掉防御层和vol三段式逻辑。100%仓位永远在进攻层（纳指/沪深300/黄金）之间轮动，选top2。

**效果**：年化**17.52%**，回撤**14.16%**，夏普**1.110**。

**原理**：13年长期牛市中，进攻层动量效应足够强。

**代价**：回撤14.16%，已超出多数个人投资者承受范围。2015年、2018年、2022年体验极差。

**适合**：年轻、风险承受能力强、投资期限10年以上的投资者。

---

#### 10.2.3 缩短动量窗口（待验证，推荐度⭐⭐）

**操作**：4周动量缩短到2~3周。

**预估效果**：年化+0.5%~+1.0%，回撤微增0.5%。

**原理**：4周动量在2020年3月V型反转、2022年10月反弹中滞后1-2周。更短窗口更快捕捉趋势转折。

**风险**：窗口过短引入噪音，需配合更高调仓阈值（7%→10%）。

**状态**：未做全参数搜索，仅方向性预估。

#### 10.2.4 增加海外资产覆盖（待验证，推荐度⭐⭐）

**操作**：进攻层加入标普500ETF（513500）和/或日经225ETF（513520）。

**预估效果**：年化+0.5%~+1.5%，回撤±1%。

**原理**：标普500（价值风格）与纳指（成长风格）轮动互补，日经提供亚太分散。

**风险**：更多资产可能稀释最强资产集中度；QDII额度和溢价问题更复杂。

**状态**：未回测验证。

### 10.3 减小回撤方向

#### 10.3.1 收紧vol防御阈值（已验证，推荐度⭐）

**操作**：`step_low`从20%降到15%，`step_high`从35%降到25%。

| 方案 | 年化 | 回撤 | 夏普 | 说明 |
|------|------|------|------|------|
| step_low=15%, step_high=25% | **12.89%** | **9.52%** | **1.046** | 收益-1.17%，回撤+1.31% |

**原理**：更早切入防御、更激进地加仓国债。

**实际效果**：与预期相反——回撤不降反升。原因是step_high=25%导致策略长期处于高防御状态，错过了大量反弹收益。2019-2021年慢牛中vol长期在20%-30%震荡，策略几乎全程防御，最终"躲过了下跌也错过了上涨"。

**结论**：收紧vol阈值**不可行**。防御过早上车反而导致组合在震荡市中持续"踏空"。

---

#### 10.3.2 启用8%净值止损（已验证触发1次，推荐度⭐⭐）

**操作**：将当前"8%止损"改为实盘强制触发规则——当周回撤触及8%时，下一周强制100%切防御层。

**实际触发**：统一引擎回测中触发**1次**（2022-11-07，nav=2.5391，peak=2.7661，回撤-8.21%）。但vol防御已提前数周行动，止损触发时防御比例已在高位，实际效果有限。

**预估效果**：极端年份回撤可能从15%压缩到10%左右，正常年份无影响。

**风险**：可能"误杀"——2020年3月第一周跌8%后切防御，错过后续反弹。

**适合**：极度厌恶回撤、宁可错杀不愿深套的投资者。

---

#### 10.3.3 趋势过滤器（待验证，推荐度⭐⭐）

**预估效果**：回撤-1%~-2%，年化-0.5%~-1%。

**原理**：长期趋势公认分界线，熊市时限制进攻仓位。

**风险**：均线滞后，震荡市产生反复信号。需配合"连续2周跌破"确认。

### 10.4 已证伪方向（经验证不可行）

以下方向经统一引擎独立回测验证，实际效果为负或劣于基准。不推荐使用。

#### 10.4.1 红利低波进进攻层（已证伪）

| 配置 | 年化 | 回撤 | vs基准 |
|------|------|------|--------|
| 红利低波进进攻层，防御仅国债 | **11.27%** | **8.60%** | **-2.79%** |

**失败原因**：
1. 红利低波年化收益远低于纳指和黄金。挤进top2进攻层就是挤掉高收益资产的位置。
2. 红利低波与沪深300相关性0.673，本质还是A股大盘。在A股熊市中它和沪深300同步下跌，无法提供真正分散。
3. 红利低波的价值是"跌得少"，不是"涨得多"。这是防御属性，不是进攻属性。

**结论**：红利低波的正确角色是**防御层**。

#### 10.4.2 现金替代国债（已证伪）

**效果**：年化**13.44%**，回撤**8.79%**（vs基准：-0.62%年化，+0.58%回撤）。

**失败原因**：现金零收益。防御层中50%仓位（平均约12.5%总仓位）从国债的~4%年化变成0%，直接拖累组合约0.5%年化。国债在利率下行期还有资本利得。

**结论**：现金是收益毒药。防御层中国债不可替代，但影响比之前预估的-4.3%要小得多（实际仅-0.62%）。

#### 10.4.3 纳指100%持仓（已证伪）

**效果**：年化**14.57%**，回撤**12.25%**，夏普**0.980**（vs基准：+0.51%年化，+4.04%回撤）。

**失败原因**：单押纳指时波动率完全暴露，2022年纳指-33%导致组合深度回撤。top2平分结构在分散化和集中度间取得最优平衡。

**意外发现**：纳指100%的年化收益（14.57%）反而高于基准（14.06%）！说明纳指在13年回测中确实是收益最强的进攻资产。但代价是回撤从8.21%飙到12.25%，夏普从1.10降到0.98。**集中持仓提升了收益，却以不成比例的回撤代价为代价。**

**结论**：不要单押任何一只ETF。夏普才是最终目标，不是年化。

- 收紧vol阈值反而使回撤从8.21%升到9.52%
- 止损兜底8%在整个13年回测中仅触发1次，且vol防御已提前行动
- 当前基准（14.06%/8.21%）本身就是收益-回撤的最优平衡点

**现实建议**：如果8.21%回撤仍超出承受范围，应考虑**降低整体仓位**（如只投入70%资金，30%留现金），而非修改策略参数。

#### 不推荐的方向：

- **❌红利低波进进攻层**：收益降2.79%，夏普下降。
- **❌现金替代国债**：收益仅降0.62%，但夏普下降。影响比预估小，但仍为负。
- **❌纳指100%**：虽然收益略增，但回撤+4%，夏普下降。集中持仓不可取。
- **❌收紧vol阈值**：收益和回撤双输，是"防御陷阱"。
- **❌动量保护**：经测试几乎无效果（消融实验已验证）。

### 10.7 关键教训：数据验证的重要性

本次修正揭示了一个核心教训：**不同引擎、不同脚本跑出的数据，即使参数相同，结果也可能差异巨大。**

v2.2消融实验中的部分数据来自早期脚本（如"去掉动量9.41%/21.71%"、"top_n=1 12.33%/18.43%"），与统一引擎回测结果（11.21%/17.24%和14.57%/12.25%）差距显著。原因可能是：
1. 早期脚本的周一对齐方式不同
2. 数据填充/缺失值处理不同
3. 手续费计算时机不同

**v2.3起，所有数据必须由统一引擎（`run_unified_v2`）在同一数据集上产生。** 任何新方向的验证，都必须经此引擎回测后才能写入文档。
